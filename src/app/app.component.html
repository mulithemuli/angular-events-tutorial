Some Event examples

All these tasks can be completed with pure angular interaction.
To take a look at EventEmitter and Subject would be a good idea.
Both of these objects make use of the publisher-subscriber pattern.

It is allowed to add @Input() and @Output attributes to the components or use a service providing a Subject to interact between components.

And by the way - it is not allowed to modify app-component or create new components (for workarounds to pass data). There exists an empty service, MessengerService which could (and probably should) be used to provide a place for the publisher-subscriber pattern.
